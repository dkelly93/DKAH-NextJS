Why?

  • File-System Routing
  • Automatic Code Splitting
  • Server Side Rendering
  • Static Exporting
  • and more...
  • Next.js will serve each file in /pages under a pathname matching the filename. (Talk more about this in a few slides)
  • Every import you declare gets bundled and served with each page. That means pages never load unnecessary code!
  • What it says on the tin. Superior performance with lightning fast delivery of content
  • Static exporting out of the box
  • Express custom server
  • Prefetching
  • Styled Components
  • Parameterized Routing
